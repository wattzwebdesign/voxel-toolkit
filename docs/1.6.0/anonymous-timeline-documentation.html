Anonymous Timeline

The Anonymous Timeline function lets users post on timelines, walls, and reviews anonymously. Each user gets a unique anonymous ID that stays consistent across their anonymous posts while keeping their identity hidden.

Overview

Sometimes users want to share feedback or participate in discussions without revealing their identity. This function adds a "Post Anonymously" option that replaces their name, avatar, and profile link with anonymous placeholders while maintaining moderation capabilities for administrators.

Key Features

  • "Post Anonymously" checkbox on timeline posts and comments
  • Unique persistent Anonymous Member ID per user
  • Works on reviews, post walls, post timelines, and user timelines
  • Administrators can still see real identities
  • Anonymous likes are also anonymized
  • Prevents reposting/quoting anonymous posts
  • Anonymous ID column in WordPress Users table

Setup Instructions

Enable the Function

  1. Navigate to Voxel Toolkit > Functions in your WordPress admin
  2. Find Anonymous Timeline in the list
  3. Toggle the switch to enable the function
  4. Click Save Changes

The "Post Anonymously" option will now appear on all timeline feeds.

How It Works

Posting Anonymously

  1. User opens the timeline post composer
  2. They check the "Post Anonymously" checkbox
  3. They write and submit their post
  4. The post displays with "Anonymous Member #12345" instead of their name

Anonymous Member ID

Each user gets a unique 5-digit Anonymous Member ID:

  • Generated when they first post anonymously
  • Stays the same for all their anonymous posts
  • Allows community members to recognize consistent anonymous contributors
  • Cannot be traced back to the user by other members

What Gets Hidden

  • Name - Replaced with "Anonymous Member #XXXXX"
  • Avatar - Replaced with generic anonymous avatar
  • Profile Link - Removed, clicking name does nothing

Administrator Access

Site administrators can always see the real identity of anonymous posters:

  • Real name shown in parentheses after anonymous ID
  • Profile link still works for admins
  • Enables proper moderation of anonymous content
  • Admin access based on user role

Anonymous Likes

When a user likes a post while they have posted anonymously:

  • Their like shows as coming from their anonymous ID
  • Maintains anonymity even for engagement

Restrictions

To prevent misuse, some actions are blocked for anonymous posts:

  • Reposting - Cannot repost an anonymous post
  • Quoting - Cannot quote an anonymous post

This prevents anonymous content from being shared in ways that could compromise the original poster's intent.

Admin Tools

Users Table Column

An "Anonymous ID" column is added to the WordPress Users table showing each user's anonymous ID (if they have one).

Deactivation Warning

When disabling this function, a warning appears that all anonymous posts will become public and show real identities. This cannot be undone.

Use Cases

Honest Reviews

Users may feel more comfortable leaving honest feedback when they can do so anonymously.

Sensitive Topics

For communities discussing sensitive topics, anonymity encourages participation.

Whistleblowing

Users can report issues or concerns without fear of identification.

Community Questions

Users can ask questions they might find embarrassing without revealing who they are.

Troubleshooting

Anonymous Option Not Showing

  • Check that the function is enabled
  • Verify you're logged in
  • Clear browser cache and refresh

Identity Still Visible

  • The "Post Anonymously" checkbox must be checked before posting
  • Existing posts cannot be made anonymous retroactively
  • Admins will always see real identities - this is by design

Anonymous ID Changed

  • Anonymous IDs are permanent once assigned
  • If it changed, the user may have a different account

Requirements

  • Voxel Theme (active)
  • Voxel Toolkit plugin (enabled)
  • Voxel timeline features in use
