Suggest Edits

The Suggest Edits function allows users to suggest corrections or updates to posts they don't own. Suggestions are submitted for review, and post owners or admins can approve or reject them.

Overview

Enable community-driven accuracy for your directory listings. Users can suggest corrections to information, upload proof images, and even mark listings as permanently closed. Suggestions go through a moderation workflow before being applied.

Key Features

  • User-submitted edit suggestions
  • Field-by-field suggestion system
  • Proof image upload support
  • "Permanently Closed" marking option
  • Admin review interface
  • Owner notification via Voxel App Events
  • Elementor widgets included
  • Bulk actions for suggestion management

Setup Instructions

Step 1: Enable the Function

  1. Navigate to Voxel Toolkit > Functions in your WordPress admin
  2. Find Suggest Edits in the list
  3. Toggle the switch to enable the function
  4. Click Save Changes

Step 2: Configure Post Types

  1. In the Post Types section, select which post types allow suggestions
  2. For each post type, select which fields can be suggested for editing
  3. Enable "Permanently Closed" option if applicable
  4. Click Save Changes

Step 3: Add Frontend Widget

  1. Edit your single listing template in Elementor
  2. Add the Suggest Edits Widget (VT)
  3. Configure button text and styling
  4. Save the template

Available Widgets

Suggest Edits Widget

Place on single post templates to show the "Suggest Edit" button.

Pending Suggestions Widget

Show pending suggestions to post owners on their account pages.

Admin Interface

Suggestions are managed in the WordPress admin:

Accessing Suggestions

  1. Go to the post type menu (e.g., Places)
  2. Click Suggested Edits submenu
  3. View all pending, approved, and rejected suggestions

Suggestion Actions

Bulk Actions

Select multiple suggestions and:

  • Bulk approve
  • Bulk reject
  • Bulk delete

Notification Integration

The function integrates with Voxel's App Events system:

Available Events

  • Suggestion Submitted: Notifies post owner when someone suggests an edit
  • Suggestion Approved: Notifies suggester when their edit is approved
  • Suggestion Rejected: Notifies suggester when their edit is rejected

Configure these events in Voxel > App Events to enable email, SMS, or in-app notifications.

Permanently Closed Feature

For business listings, users can suggest marking a location as permanently closed:

  • Special "Permanently Closed" checkbox in suggestion form
  • When approved, post can be moved to a special status
  • Useful for directories with business listings

Use Cases

Business Directories

Let users report outdated business information like wrong hours or addresses.

Event Listings

Allow attendees to suggest corrections to event details.

Community Content

Crowdsource accuracy improvements for community-generated content.

Troubleshooting

Widget Not Showing

  • Ensure the function is enabled
  • Verify the post type is configured for suggestions
  • Check that at least one field is enabled for suggestions
  • Widget may be hidden from post owners (they can edit directly)

Suggestions Not Saving

  • Check database table exists (vx_toolkit_suggestions)
  • Verify user is logged in
  • Check browser console for JavaScript errors

Admin Menu Not Appearing

  • Ensure the post type is public
  • The submenu appears under each configured post type
  • Verify user has permission to manage posts

Requirements

  • Voxel Theme (active)
  • Voxel Toolkit plugin (enabled)
  • Elementor (for frontend widgets)
  • Users must be logged in to submit suggestions
