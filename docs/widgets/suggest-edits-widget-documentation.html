Suggest Edits Widget

The Suggest Edits widget allows visitors to suggest corrections or updates to listing information. Users can propose changes to specific fields, upload proof photos, and mark listings as permanently closed.

Overview

Enable community-driven data accuracy by letting users suggest edits to post fields. The widget displays a button that opens a modal showing current values and input fields for suggested changes. Submissions are stored for admin review before being applied.

Key Features

  • Field-by-field edit suggestions
  • Current value display for comparison
  • Photo upload for proof of changes
  • "Permanently closed" option
  • "Don't know but incorrect" checkbox
  • Taxonomy and select field support
  • Work hours editor
  • Guest submission support

Setup Instructions

Step 1: Enable the Function

  1. Navigate to Voxel Toolkit > Functions in your WordPress admin
  2. Find Suggest Edits in the list
  3. Toggle the switch to enable
  4. Configure which post types can receive suggestions
  5. Optionally enable guest submissions
  6. Click Save Changes

Step 2: Add Widget to Template

  1. Edit your single listing template in Elementor
  2. Search for Suggest Edits (VT)
  3. Drag the widget to your desired location
  4. Select which fields users can suggest edits for
  5. Save the template

Supported Field Types

The widget supports editing these Voxel field types:

  • Text fields
  • Textarea/Description
  • Number fields
  • Select/Multiselect fields
  • Taxonomy fields
  • Location fields
  • Work hours fields
  • Image fields
  • Rich text/Editor fields

Excluded field types: timezone, switcher, file, repeater, recurring-date, post-relation, time, and UI fields.

Work Hours Editor

For work hours fields, a special popup editor appears allowing users to:

  • View current schedule grouped by days
  • Add new schedule groups
  • Set open/closed status for each group
  • Configure multiple time slots per day
  • Save suggested hours

Guest Submissions

When guest submissions are enabled:

  • Name and email fields appear in the modal
  • Email is required for submission
  • Guest data is stored with the suggestion

Use Cases

Business Directories

Let visitors suggest updated hours, phone numbers, or addresses for local businesses.

Restaurant Listings

Allow customers to report menu changes, new hours, or closures.

Event Venues

Enable users to suggest capacity updates or amenity changes.

Real Estate

Let visitors report sold properties or price changes.

Reviewing Suggestions

Suggestions are stored and can be reviewed in:

  1. Navigate to Voxel Toolkit > Pending Suggestions
  2. View all pending suggestions by post
  3. Compare current vs suggested values
  4. Approve or reject each suggestion
  5. View uploaded proof photos

Troubleshooting

Widget Not Showing

  • Verify Suggest Edits function is enabled
  • Check current post type is in allowed list
  • Ensure fields are selected in widget settings

No Fields Available

  • Widget only shows fields with values
  • Empty fields are automatically excluded
  • Some field types (UI, repeater) are excluded

Submission Fails

  • At least one change must be made
  • Guests must provide email
  • Check browser console for AJAX errors

Requirements

  • Voxel Theme (active)
  • Voxel Toolkit plugin (enabled)
  • Elementor (for widget placement)
  • Suggest Edits function enabled
  • Post type with editable Voxel fields
